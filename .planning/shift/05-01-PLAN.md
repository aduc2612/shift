### Task 2: Create QueryProvider

**File:** `src/providers/query-provider.tsx`

- Create `QueryClient` with sensible defaults
- Wrap children with `QueryClientProvider`
- Export `useQueryClient` for cache invalidation

```tsx
const queryClient = new QueryClient({
  defaultOptions: {
    queries: {
      staleTime: 30_000, // 30s
      retry: 2,
    },
  },
});

export function QueryProvider({ children }: { children: React.ReactNode }) {
  return (
    <QueryClientProvider client={queryClient}>{children}</QueryClientProvider>
  );
}
```

### Task 4: Create task API functions

**File:** `src/features/schedule/api.ts`

Pure async functions that call Supabase. No hooks, no state — just data fetching.

```ts
export async function fetchTasks(userId: string, date: Date): Promise<Task[]>
export async function createTask(task: Omit<Task, 'id' | 'createdAt' | 'updatedAt'>): Promise<Task>
export async function updateTask(id: string, updates: Partial<Task>): Promise<Task>
export async function deleteTask(id: string): Promise<void>
export async function toggleTaskComplete(id: string, completed: boolean): Promise<Task>
```

- `fetchTasks` filters by `user_id` and date (using `startTime` within day range)
- All functions use `supabase.from('tasks')` queries
- Return typed results, throw on error

### Task 5: Create task hooks

**File:** `src/features/schedule/hooks/useTasks.ts`

TanStack Query hooks wrapping API functions.

```ts
export function useTasks(date: Date)          // useQuery
export function useCreateTask()               // useMutation
export function useUpdateTask()               // useMutation
export function useDeleteTask()               // useMutation
export function useToggleComplete()           // useMutation with optimistic update
```

- All mutations invalidate `['tasks']` query key
- `useToggleComplete` uses `onMutate` for optimistic update, `onError` for rollback
- `useCreateTask` optimistically adds to cache
- `useDeleteTask` optimistically removes from cache

### Task 6: Create custom Alert primitive

**File:** `src/components/primitives/Alert.tsx`

Modal-based alert dialog. No React Native `Alert`.

**Props:**
```ts
type AlertProps = {
  visible: boolean;
  title: string;
  message?: string;
  confirmLabel?: string;  // default "Confirm"
  cancelLabel?: string;   // default "Cancel"
  destructive?: boolean;  // true = red confirm button
  onConfirm: () => void;
  onCancel: () => void;
};
```

**Implementation:**
- `Modal` transparent, centered content
- Backdrop: semi-transparent `scrim` color
- Card: `surface` background, rounded corners, padding
- Title: `titleMedium`, `onSurface`
- Message: `bodyMedium`, `onSurfaceVariant`
- Buttons: row at bottom — Cancel (surfaceVariant) + Confirm (primary or error if destructive)
- Uses theme tokens via `createStyles(theme)` pattern
