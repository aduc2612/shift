### Task 12: Build the Schedule screen (`src/app/(tabs)/index.tsx`)

Replace the placeholder with the full schedule screen.

**State:**
- `selectedDate: Date` (defaults to today)
- `showDatePicker: boolean`
- `showRescheduleSheet: boolean`

**Layout:**
1. Safe area top padding
2. `ScheduleHeader` — date + reschedule button
3. `ScheduleProgress` — progress bar + meta
4. FlatList with discriminated union items: `{ type: 'task'; task: Task } | { type: 'now' }`
5. `FAB` — absolute positioned bottom-right
6. `DateTimePicker` — shown when date label tapped
7. `BottomSheet` — reschedule sheet (pure UI for now)

**Date picker:**
- Uses `@expo/ui/community/datetime-picker` with `presentation="inline"`
- No platform branching — same component on iOS and Android
- Show/hide by conditional rendering (`{showDatePicker && <DateTimePicker ... />}`)
- `onValueChange` only fires on user interaction (no mount-time phantom event)
- When date changes: update `selectedDate`, which filters the task list

**Task filtering:**
- Tasks filtered by `isSameDay(new Date(task.startTime), selectedDate)`
- Empty state: "No tasks for this day"

**Task state logic:**
- `task.completed === true` → `done` (visual strikethrough + reduced opacity)
- `now >= start && now < end` → `active` (highlighted card)
- Everything else → `upcoming` (normal card)
- A task whose time has passed but is NOT checked off remains `upcoming` (no auto-strikethrough)

**Now indicator placement:**
- If an active task exists: the first active task gets `showNow={true}` on its TimelineRow (now dot on its spine)
- If no active task exists: a standalone `{ type: 'now' }` row is inserted between tasks whose time has passed (`endTime <= now`) and upcoming tasks
- Only one now indicator visible at a time
- `buildScheduleData` returns both `{ items, activeTaskId }` in one pass (no duplicate computation)

**Reschedule sheet:**
- Extracted as `RescheduleSheet.tsx` — wraps `BottomSheet`, manages its own text state
- Title: "What changed?"
- `TextInput` multiline with placeholder
- Example chips (static list): "I woke up late", "I need to leave at 5 PM", "Gym is closed today", "Assignment is more important"
- "Reschedule" CTA button (primary color, full width)
- CTA is no-op for now (wired in Phase 6)

**FAB:**
- `onPress`: placeholder (no-op for now, will open AddTaskSheet in Phase 5)

**Mock data for development:**
- `src/features/schedule/mockTasks.ts` with 5 sample tasks

## Notes

- **No monospace:** Using default system font (Inter on Android, San Francisco on iOS) for all text.
- **DateTimePicker:** Uses `@expo/ui/community/datetime-picker` (Expo drop-in). No platform branching. `presentation="inline"` on both platforms.
- **Now indicator pulse:** `Animated.loop` with `Animated.sequence` for opacity pulse (0.4 ↔ 1 over 2s).
- **Now indicator time:** Updates every 30s via `setInterval` in both `TimelineRow` (embedded) and `NowIndicator` (standalone).
- **FlatList data union:** `{ type: 'task'; task: Task } | { type: 'now' }` for the FlatList items.
- **Task state:** `done` only when `task.completed === true`. Time passing does NOT auto-complete.
- **Now insertion:** Uses `task.endTime <= now` for positioning (not visual state), so unchecked-but-passed tasks correctly appear before the now indicator.
- **Now time:** Uses `useCurrentTime` hook (30s interval). Screen's `now` value updates live.
- **Business logic extracted:** `getTaskState`, `buildScheduleData` live in `src/features/schedule/utils.ts`.
- **Color opacity:** `withOpacity(hex, alpha)` in `src/utils/color.ts` — used by Badge (accent bg) and TaskCard (active border).
- **RescheduleSheet:** Standalone component wrapping BottomSheet. CTA button is no-op for now.
- **TimelineRow:** Merged duplicate dot styles into `dotHighlighted`. Animation value created only when `showNow`.
- **Bottom sheet:** Primitive with `animationType="none"`, theme `scrim` backdrop. Keyboard padding delegated to consumers via `useKeyboardHeight` hook.
- **Theme scrim:** `#00000066` (40% opacity) for modal overlays.
- **No Supabase yet:** Tasks come from mock data.
