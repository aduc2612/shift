# Phase 7 — Push Notifications

## Objective

Implement local scheduled notifications for task reminders. Each incomplete task gets 4 notifications: 10 min before start, on task start, on task end, and a nudge 5 min after end (cancelled if task completed). All notifications resynced from scratch on every trigger. Notification tap opens app to schedule screen.

## Context

- Phases 1–6 complete: project structure, auth, Sentry, schedule UI, task CRUD, AI reschedule all working
- `src/services/notifications.ts` exists — placeholder `syncNotifications` function
- `expo-notifications` already installed (`~56.0.6`)
- `expo-constants` already installed
- TanStack Query hooks: `useTasks`, `useCreateTask`, `useUpdateTask`, `useToggleComplete` all invalidate `['tasks']` query key
- `useReschedule` hook exists — invalidates queries on success
- Tasks always have `startTime` and `endTime` (AI assigns immediately)
- No permission request in this phase — deferred to Phase 8 (onboarding)
- Notification tap just opens app (no deep linking to specific task)

**Key decisions:**

- **Local notifications only** — `Notifications.scheduleNotificationAsync` with time triggers. No push tokens, no server-side sending.
- **TDD approach** — write tests first, then implement
- **Single Android channel** — "Task Reminders" for all notification types
- **`syncNotifications` is the only place** that schedules or cancels notifications (per AGENTS.md)
- **Always cancel all first** — `cancelAllScheduledNotificationsAsync` then rewrite from scratch
- **Skip nudge for completed tasks** — if task is already complete at sync time, don't schedule its nudge

```
App                         expo-notifications
 │                                │
 ├─ syncNotifications(tasks)      │
 │  ├─ cancelAllScheduled()  ────►│  clears everything
 │  ├─ for each incomplete task:  │
 │  │  ├─ schedule "10 min before" ──►│ trigger: date = startTime - 10min
 │  │  ├─ schedule "task start"    ──►│ trigger: date = startTime
 │  │  ├─ schedule "on end"       ──►│ trigger: date = endTime
 │  │  └─ schedule "nudge"        ──►│ trigger: date = endTime + 5min
 │  └─ done                       │
 │                                │
 ├─ User taps notification        │
 │  └─ Opens app (schedule screen)│
 │                                │
 ├─ User completes task           │
 │  └─ syncNotifications again    │  (nudge for that task now skipped)
```

#### Step 2: Implement

**File:** `src/services/notifications.ts`

```ts
import * as Notifications from "expo-notifications";
import { Platform } from "react-native";
import type { Task } from "@/types/task";

const CHANNEL_ID = "task-reminders";

// Show notification even when app is in foreground
Notifications.setNotificationHandler({
  handleNotification: async () => ({
    shouldShowAlert: true,
    shouldPlaySound: true,
    shouldSetBadge: false,
  }),
});

export async function setupNotificationChannel(): Promise<void> {
  if (Platform.OS === "android") {
    await Notifications.setNotificationChannelAsync(CHANNEL_ID, {
      name: "Task Reminders",
      importance: Notifications.AndroidImportance.HIGH,
      vibrationPattern: [0, 250, 250, 250],
      lightColor: "#FF231F7C",
    });
  }
}

export async function syncNotifications(tasks: Task[]): Promise<void> {
  await Notifications.cancelAllScheduledNotificationsAsync();

  const now = new Date();

  for (const task of tasks) {
    if (task.completed) continue;

    const start = new Date(task.startTime);
    const end = new Date(task.endTime);
    const tenBefore = new Date(start.getTime() - 10 * 60 * 1000);
    const fiveAfter = new Date(end.getTime() + 5 * 60 * 1000);

    // 10 min before start
    if (tenBefore > now) {
      await Notifications.scheduleNotificationAsync({
        content: {
          title: `${task.name} starts soon`,
          body: `Starting in 10 minutes`,
          data: { taskId: task.id },
          ...(Platform.OS === "android" ? { channelId: CHANNEL_ID } : {}),
        },
        trigger: { date: tenBefore },
      });
    }

    // On task end
    if (end > now) {
      await Notifications.scheduleNotificationAsync({
        content: {
          title: `Time is up — ${task.name}`,
          body: `Your scheduled time for this task has ended`,
          data: { taskId: task.id },
          ...(Platform.OS === "android" ? { channelId: CHANNEL_ID } : {}),
        },
        trigger: { date: end },
      });
    }

    // Nudge 5 min after end
    if (fiveAfter > now) {
      await Notifications.scheduleNotificationAsync({
        content: {
          title: `Haven't finished ${task.name}?`,
          body: `It's been 5 minutes since your scheduled time ended`,
          data: { taskId: task.id },
          ...(Platform.OS === "android" ? { channelId: CHANNEL_ID } : {}),
        },
        trigger: { date: fiveAfter },
      });
    }
  }
}
```

#### Step 2: Implement

**File:** `src/features/schedule/hooks/useSyncNotifications.ts`

```ts
import { useEffect } from "react";
import { useQueryClient } from "@tanstack/react-query";
import { syncNotifications } from "@/services/notifications";
import type { Task } from "@/types/task";

/**
 * Observes the TanStack Query cache for task data changes
 * and calls syncNotifications whenever tasks are refetched.
 */
export function useSyncNotifications() {
  const queryClient = useQueryClient();

  useEffect(() => {
    const unsubscribe = queryClient.getQueryCache().subscribe((event) => {
      if (event.type === "updated" && event.query.queryKey[0] === "tasks") {
        const data = event.query.state.data as Task[] | undefined;
        if (data) {
          syncNotifications(data).catch((err) => {
            console.error("Failed to sync notifications:", err);
          });
        }
      }
    });

    return unsubscribe;
  }, [queryClient]);
}
```

## Notes

- **Local notifications only** — no push tokens, no Expo push service, no server-side sending. All scheduling happens on-device via `expo-notifications`.
- **No permission request in this phase** — deferred to Phase 8. If permission is not granted, `scheduleNotificationAsync` will fail silently (we don't catch per-notification, but the whole sync is wrapped in a catch in the hook).
- **Sync is cheap** — cancel all + rewrite is O(n) for n tasks. With <100 tasks, this is instant.
- **Query cache observer pattern** — `useSyncNotifications` subscribes to TanStack Query cache updates. This means notifications sync on every successful fetch, mutation, and invalidation. No need to manually call `syncNotifications` from individual hooks.
- **Foreground notifications** — `setNotificationHandler` configured to show alerts even when app is open. User sees notification banner while using the app.
- **Task data in notification payload** — `taskId` included in `data` field for future use (e.g., deep linking to task detail in Phase 10 polish).
