## Tasks

### 1. Initialize Expo project
- Run `npx create-expo-app@latest . --template blank-typescript` (or use `tabs` template as starting point for Expo Router)
- If the tabs template includes boilerplate we don't need, strip it down
- Ensure TypeScript strict mode is enabled in `tsconfig.json`
- Verify the app starts with `npx expo start`

### 2. Install required dependencies
Core dependencies:
- `expo-router`
- `react-native-safe-area-context`
- `react-native-screens`
- `@supabase/supabase-js`
- `zustand`
- `react-native-dotenv` (for `@env` imports)
- `expo-notifications`
- `expo-linking`
- `expo-constants`
- `expo-status-bar`

Dev dependencies:
- `@types/react`
- `typescript`

Install via: `npx expo install <packages>` (prefer expo install for compatibility)

### 3. Configure Expo Router
- Update `package.json` with `"main": "expo-router/entry"`
- Configure `app.json` / `app.config.js`:
  - Set `"scheme": "shift"` (for deep linking)
  - Set `"plugins": ["expo-router"]`
- Create `app/_layout.tsx` as root layout
- Delete any default template files not needed

### 4. Create folder structure
Create the following directories inside `src/` (or at root if Expo Router requires `app/` at root):

```
app/                    # Expo Router routes (must be at root)
  _layout.tsx           # Root layout with SafeAreaProvider + ThemeProvider
  (auth)/
    _layout.tsx         # Auth group layout (placeholder)
    index.tsx           # Redirect/placeholder
  (tabs)/
    _layout.tsx         # Tab navigator layout
    index.tsx           # Schedule screen (placeholder)
    settings.tsx        # Settings screen (placeholder)
components/
constants/
  theme.ts
  images.ts
data/
hooks/
lib/
  supabase.ts
  ai.ts                 # Empty placeholder
  notifications.ts      # Empty placeholder
  revenuecat.ts         # Empty placeholder
store/
types/
  task.ts
  userPreferences.ts
assets/
  images/               # (empty for now)
providers/
  theme-provider.tsx
```

Note: `app/` must be at project root for Expo Router. Other folders (`components/`, `lib/`, etc.) go at project root too for clean imports with `@/` alias.

### 5. Configure path aliases
- Update `tsconfig.json` with `"baseUrl": "."` and `"paths": { "@/*": ["./*"] }`
- Update `babel.config.js` with `module-resolver` plugin for `@/` alias
- Or use Expo Router's built-in `@/` convention if available

### 6. Create theme system (`constants/theme.ts`)
- Define color tokens for light and dark modes
- Define spacing scale (xs, sm, md, lg, xl)
- Define typography scale (sizes, weights)
- Define border radius tokens
- Define shadow tokens
- Export `createTheme(isDark: boolean)` function
- Export `Theme` type
- Reference `constants/ThemeExampleStylesheet.tsx` if it exists

### 7. Create ThemeProvider (`providers/theme-provider.tsx`)
- React context that provides current theme
- Detects system color scheme via `useColorScheme()`
- Exports `useTheme()` hook
- Wraps children with theme context

### 8. Set up root layout (`app/_layout.tsx`)
- Wrap app in `SafeAreaProvider` (from `react-native-safe-area-context`)
- Wrap app in `ThemeProvider`
- Set up `Slot` or `Stack` for navigation
- Set status bar style based on theme

### 9. Set up tab navigator (`app/(tabs)/_layout.tsx`)
- Use `Tabs` from `expo-router`
- Two tabs:
  - Schedule (icon: calendar or list)
  - Settings (icon: gear)
- Tab bar styled with theme colors

### 10. Create placeholder screens
- `app/(tabs)/index.tsx` â€” Schedule screen with centered "Schedule" text
- `app/(tabs)/settings.tsx` â€” Settings screen with centered "Settings" text
- `app/(auth)/_layout.tsx` â€” Simple Stack layout
- `app/(auth)/index.tsx` â€” Placeholder "Auth" text

### 11. Configure Supabase client (`lib/supabase.ts`)
- Import `createClient` from `@supabase/supabase-js`
- Import `SUPABASE_URL` and `SUPABASE_ANON_KEY` from `@env`
- Export configured `supabase` client

### 12. Create `.env` file
- Add placeholder values:
  ```
  SUPABASE_URL=https://your-project.supabase.co
  SUPABASE_ANON_KEY=your-anon-key
  ```
- Add `.env` to `.gitignore`
- Create `.env.example` with same keys (committed to repo)

### 13. Create type definitions (`types/`)
- `types/task.ts` â€” Task type matching AGENTS.md schema
- `types/userPreferences.ts` â€” UserPreferences type matching AGENTS.md schema

### 14. Create empty lib placeholders
- `lib/ai.ts` â€” Export empty `rescheduleTasks()` function stub
- `lib/notifications.ts` â€” Export empty `syncNotifications()` function stub
- `lib/revenuecat.ts` â€” Export placeholder comments

### 15. Update `.gitignore`
- Ensure `.env` is ignored
- Ensure standard Expo/Node ignores are present

### 16. Clean up template boilerplate
- Remove any default template components/screens not needed
- Remove unused assets
- Verify no unused imports
